# Montage Skill

Assemble multiple generated video clips into a single final video using ffmpeg.

## When to Use

- After completing a long-video pipeline -- all shots generated, user wants final assembly
- When user explicitly asks to stitch, combine, merge, or montage multiple video clips
- Pipeline-driven from a UGC workflow (ugc-flow, ugc-tutorial-flow, ugc-try-on-flow, ugc-unboxing-flow)
  when N >= 2 clips are present in output/ named clip_1.mp4 ... clip_N.mp4.
  Hard-cut concat in numeric board order to output/final.mp4. No user confirmation needed.
- NOT for single video generation -- that's /video-generation
- NOT automatically after every generation -- only when user requests or pipeline completes

## When NOT to Use

- User generated 10 separate videos as individual pieces -- no stitching needed
- User hasn't finished generating all clips yet
- User wants to edit a single video (trim, crop, etc.)

## Transition Rule

Default: hard cut. All clips are joined via direct concatenation with no crossfade, dissolve, wipe,
or any other transition effect. AI-generated clips already contain internal motion; adding software
transitions creates visual artifacts and breaks immersion. The cut itself is the transition.

Transitions allowed ONLY by explicit user request. Without such a request -- always hard cut.
Never add transitions by default, never suggest them proactively.

## Assembly Methods

### 1. Concatenation (hard cut)

```bash
# Create file list
for f in output/videos/shot_*.mp4; do
  echo "file '$f'" >> output/filelist.txt
done

# Concatenate -- hard cut, stream copy (no re-encode)
ffmpeg -f concat -safe 0 -i output/filelist.txt -c copy output/final.mp4
```

### 2. Add Background Music

```bash
ffmpeg -i output/final.mp4 -i bgm.mp3 \
  -c:v copy -c:a aac -shortest output/final_with_bgm.mp4
```

### 3. Add Subtitles (SRT)

```bash
ffmpeg -i output/final.mp4 -vf subtitles=output/subs.srt output/final_subtitled.mp4
```

## Pipeline-Specific Assembly

### Long Video
- Read assembly.json from output/ -- it has shot order, BGM notes
- Follow the assembly plan exactly
- Always use hard-cut concatenation

## Quality Rule

Output resolution = source resolution. The final video must match the resolution and quality
of the input clips exactly. Use -c copy (stream copy) wherever possible to preserve original
quality byte-for-byte. Never upscale, never downscale, never re-encode unless absolutely
required (e.g. subtitle burn-in).

## Output

- Final video saved to output/final.mp4 (or output/final_epNNN.mp4 for episodes)
- Always verify output exists and is playable before reporting to user
- Report duration, resolution, file size

## Prerequisites

- ffmpeg must be installed on the system
- All source clips must exist in output/videos/
- All clips should have matching resolution and frame rate (they will if generated by the same pipeline)
